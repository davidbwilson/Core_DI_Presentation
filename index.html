Why would I want to use Dependency Injection?

  • Decreased coupling
  • Provides testing seams
  • Improved object composability
  • Makes decomposing tasks easier for teams
  • It's an improvement, but like everything there are trade-offs

What is a Dependency Injection Container?

  • A framework that manages dependencies for your application
  • Wires-up the entire dependency graph for a top-level object
  • Manages object life times and garbage collection
  • Several popular options in .NET

How does ASP.NET MVC implement DI?

  • Built-in / light-weight container
  • Entire framework is built on DI
  • Allows injection into views (_ViewImports.cshtml for global)
  • Service Locator Provided on HTTPContext (warning: use with caution)

What are lifetime scopes?

  • Determines the instance of an object used to fulfill a dependency
  • Available scopes in MVC:
    • Transient: Instance per dependency request
    • Scoped: Instance per HTTP request
    • Singleton: Instance for the application
    • Instance: Instance for the application created at startup

How do I make my registrations generic?

Assembly scanning allows generic registrations
dotnet add package scrutor

						services.Scan(scan => scan.FromAssemblyOf()
							.AddClasses(cl => cl.Where(tname => tname.Name.EndsWith("Service")))
							.AsImplementedInterfaces()
							.WithTransientLifetime());

Why would I replace the default MVC IoC Container?

  • Improved assembly scanning
  • Complex scoping
  • Registration Modules
  • Alternative Injection Methods
